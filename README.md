# Орбитекс | Frontend

## О проекте

Это фронтенд-прототип онлайн-сервиса для визуализации пролётов спутников и анализа орбитальных данных на основе TLE (Two-Line Element Set).

Приложение предоставляет интерактивную карту Земли, на которой спутники отображаются в реальном времени или в режиме симуляции времени. Пользователь может отслеживать движение объектов по орбитам, анализировать спутниковые группировки, применять фильтры и получать информацию о пролётах над выбранными регионами.

---

## Цель проекта

Создать удобный и наглядный веб-интерфейс для:

- визуализации движения спутников по TLE-данным;
- анализа орбит и пролётов;
- работы со спутниковыми группировками;
- фильтрации спутников по различным параметрам;
- отображения пролётов над интересующими регионами.

---

## Основные задачи фронтенда

### 1. Интерактивная карта Земли

- отображение карты Земли в 2D или 3D;
- визуализация текущего положения спутников;
- отображение траекторий и орбит;
- масштабирование, перемещение и выбор области интереса.

### 2. Визуализация спутников

- отображение спутников на карте в реальном времени;
- режим симуляции времени;
- обновление позиции объектов на основе TLE-данных;
- отображение направления движения и орбитального следа.

### 3. Аналитика и фильтрация

- фильтрация спутников по:
  - названию;
  - NORAD ID;
  - типу/назначению;
  - высоте орбиты;
  - наклонению;
  - принадлежности к группировке;
  - статусу активности;
- поиск спутника по имени или идентификатору;
- просмотр параметров выбранного спутника.

### 4. Анализ пролётов

- выбор региона или точки на карте;
- отображение ближайших пролётов над выбранной областью;
- просмотр времени начала и конца пролёта;
- получение дополнительной информации о спутнике во время пролёта.

---

## Ключевые пользовательские сценарии

- Пользователь открывает карту и видит спутники, движущиеся по орбитам.
- Пользователь выбирает спутник и получает его параметры.
- Пользователь включает симуляцию времени для анализа движения.
- Пользователь фильтрует спутники по группировке или характеристикам.
- Пользователь выбирает регион и смотрит пролёты спутников над ним.

## Предполагаемый функционал интерфейса

### Главный экран

- карта Земли;
- панель управления временем;
- панель фильтров;
- список спутников;
- информационная карточка выбранного спутника.

### Панель управления временем

- старт/пауза симуляции;
- ускорение/замедление времени;
- переход к текущему времени;
- выбор даты и времени.

### Панель фильтров

- поиск по названию;
- фильтр по орбитальным параметрам;
- фильтр по группировкам;
- фильтр по регионам пролёта.

### Информационная карточка спутника

- название;
- идентификатор;
- TLE-данные;
- высота;
- скорость;
- наклонение орбиты;
- период обращения;
- прогноз ближайших пролётов.

## Установка и запуск

1. Установите зависимости:
   ```bash
   npm ci
   ```
2. Для локальной разработки используйте:
   ```bash
   npm run dev
   ```
   сборка стартует Vite dev-сервер на `localhost:5173`.
3. Для продакшен-сборки:
   ```bash
   npm run build
   npm run preview
   ```
   Это позволит проверить результат `npm run build` через Vite preview на `localhost:4173`.

## Технологический стек

Реальный стек и библиотеки, уже внедрённые в проект:

- **React** + **Vite** + **TypeScript** для клиентского SPA.
- **Tailwind CSS** / PostCSS + утилиты в `shared/ui` для стилистики.
- **@tanstack/react-router** + **Zustand** для маршрутов и управления состоянием.
- **@react-three/fiber** + **drei** для WebGL-глобуса и 3D-рендеринга.
- **D3** (`geoPath`, `geoEqualEarth`, `geoGraticule`) для векторных карт и геометрии.
- **satellite.js** и собственный модуль `entities/satellite/lib/propagation` для орбитальных расчётов.
- `fetch` / `axios` стилизованы (см. features/simulation) для API-запросов и websocket-интеграций.
- `npm run dev|build|preview`, Dockerfile и docker-compose для локального/продакшен-развёртывания.

## Архитектура фронтенда

```
src/
  app/                      – Vite entrypoint, глобальные стили, env-config (`enviroment.ts`).
  features/
    earth-map-2d/           – SVG/D3-карта и её панель управления.
    earth-globe-3d/         – WebGL-глобус (`@react-three/fiber` + `drei`).
    tracked-satellites/      – список и карточки спутников, работа отслеживаемыми спутниками.
    simulation/              – контроллер времени и UI-панель.
  entities/
    satellite/               – доменные модели, mock-данные (+TLE), утилиты (`propagation.ts`), UI-карточки.
  widgets/
    header/, sidebar/        – повторно используемые оболочки интерфейса.
  shared/
    components/              – рисуемые кнопки, бейджи, лейауты.
    lib/                     – helpers, типы и утилиты.
    styles/                  – глобальные CSS/Tailwind.
```

Дополнительно:

- Каждый фичевый модуль делится на `ui/` (компоненты) и `model.ts`/`store.ts` (Zustand).
- Геоданные обрабатываются через `satellite.js` и переиспользуются везде (карты, 3D и карточки).
- `TrackedSatellites` читают `simulationTime` и обновляют карточки с реальными координатами и скоростью.

## Переменные окружения

Фронтенд читает переменные `VITE_PUBLIC_*` из `.env.*` файлов. Конкретную сборку можно определить через:

- `.env.development` (`VITE_PUBLIC_ENV=dev`)
- `.env.prod` (`VITE_PUBLIC_ENV=prod`)

Файл `src/app/env/enviroment.ts` экспортирует:

- `APP_ENVIRONMENT` (`"dev"` или `"prod"`) — источник правды для UI/логики в рантайме.
- `IS_DEV_ENV` / `IS_PROD_ENV` — булевые флаги.

Это позволяет компонентам быстро выяснять, под какой конфигурацией они запущены.

Дополнительно:

- `VITE_PUBLIC_SATELLITE_DATA_SOURCE` — если задать `mock`, на карте используются только локальные моки (`satellitesMapMock`). Иначе данные берутся с API `GET /tle/demo` с учётом фильтров в сайдбаре.

## Docker и docker-compose

В проект добавлено:

- `Dockerfile` со сборкой `npm run build` и финальным контейнером на nginx.
- `docker-compose.yml` для двух сред:
  - `app-dev` — монтирует локальный код, прокидывает `.env.development`, стартует dev-сервер.
  - `app-prod` — использует продакшен-образ и `.env.prod`, служит готовую `dist` через nginx.

Запуск:

```bash
docker compose up --build app-dev
# или
docker compose up --build app-prod
```
