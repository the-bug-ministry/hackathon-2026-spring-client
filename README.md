# Орбитекс | Frontend

## О проекте

Это фронтенд-прототип онлайн-сервиса для визуализации пролётов спутников и анализа орбитальных данных на основе TLE (Two-Line Element Set).

Приложение предоставляет интерактивную карту Земли, на которой спутники отображаются в реальном времени или в режиме симуляции времени. Пользователь может отслеживать движение объектов по орбитам, анализировать спутниковые группировки, применять фильтры и получать информацию о пролётах над выбранными регионами.

---

## Цель проекта

Создать удобный и наглядный веб-интерфейс для:

- визуализации движения спутников по TLE-данным;
- анализа орбит и пролётов;
- работы со спутниковыми группировками;
- фильтрации спутников по различным параметрам;
- отображения пролётов над интересующими регионами.

---

## Основные задачи фронтенда

### 1. Интерактивная карта Земли

- отображение карты Земли в 2D или 3D;
- визуализация текущего положения спутников;
- отображение траекторий и орбит;
- масштабирование, перемещение и выбор области интереса.

### 2. Визуализация спутников

- отображение спутников на карте в реальном времени;
- режим симуляции времени;
- обновление позиции объектов на основе TLE-данных;
- отображение направления движения и орбитального следа.

### 3. Аналитика и фильтрация

- фильтрация спутников по:
  - названию;
  - NORAD ID;
  - типу/назначению;
  - высоте орбиты;
  - наклонению;
  - принадлежности к группировке;
  - статусу активности;
- поиск спутника по имени или идентификатору;
- просмотр параметров выбранного спутника.

### 4. Анализ пролётов

- выбор региона или точки на карте;
- отображение ближайших пролётов над выбранной областью;
- просмотр времени начала и конца пролёта;
- получение дополнительной информации о спутнике во время пролёта.

---

## Ключевые пользовательские сценарии

- Пользователь открывает карту и видит спутники, движущиеся по орбитам.
- Пользователь выбирает спутник и получает его параметры.
- Пользователь включает симуляцию времени для анализа движения.
- Пользователь фильтрует спутники по группировке или характеристикам.
- Пользователь выбирает регион и смотрит пролёты спутников над ним.

---

## Предполагаемый функционал интерфейса

### Главный экран

- карта Земли;
- панель управления временем;
- панель фильтров;
- список спутников;
- информационная карточка выбранного спутника.

### Панель управления временем

- старт/пауза симуляции;
- ускорение/замедление времени;
- переход к текущему времени;
- выбор даты и времени.

### Панель фильтров

- поиск по названию;
- фильтр по орбитальным параметрам;
- фильтр по группировкам;
- фильтр по регионам пролёта.

### Информационная карточка спутника

- название;
- идентификатор;
- TLE-данные;
- высота;
- скорость;
- наклонение орбиты;
- период обращения;
- прогноз ближайших пролётов.

## Технологический стек

Рекомендуемый стек для фронтенда:

- **React** / **Next.js**
- **TypeScript**
- **Mapbox GL JS**, **Leaflet** или **CesiumJS** для карты и геовизуализации
- **Tailwind CSS** / **SCSS** для стилизации
- **Zustand**, **Redux Toolkit** или Context API для управления состоянием
- **Axios** / **Fetch API** для запросов к backend API
- **satellite.js** для расчётов по TLE на клиенте или предварительной обработки
